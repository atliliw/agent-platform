### 2.2 流式对话流程


## 3. 多 Agent 数据流

### 3.1 Master-Worker 模式

```
用户发送复杂任务
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 1. Gateway → Chat Service                                                   │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 2. Master Agent 分析                                                        │
│    • 系统提示词: "你是一个任务规划者..."                                     │
│    • 分析用户请求                                                            │
│    • 输出 JSON:                                                             │
│      {                                                                      │
│        "need_decompose": true,                                             │
│        "subtasks": [                                                       │
│          {"agent": "researcher", "task": "搜索相关资料"},                   │
│          {"agent": "analyst", "task": "分析数据", "depends_on": [0]},      │
│          {"agent": "writer", "task": "撰写报告", "depends_on": [0, 1]}     │
│        ]                                                                   │
│      }                                                                     │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 3. 任务分解与调度                                                           │
│    • 构建 DAG (有向无环图)                                                   │
│    • 拓扑排序确定执行顺序                                                    │
│    • 并行执行无依赖的任务                                                    │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ├──┬──────────────────────────────────────────────────┐
    │  │                                                  │
    ▼  ▼                                                  ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ Specialist #1   │  │ Specialist #2   │  │ Specialist #3   │
│ (Researcher)    │  │ (Analyst)       │  │ (Writer)        │
│                 │  │ (等待 #1)       │  │ (等待 #1, #2)   │
│ • 调用工具      │  │                 │  │                 │
│ • 调用知识库    │  │ • 调用工具      │  │ • 调用 LLM      │
│ • 返回结果      │  │ • 返回结果      │  │ • 返回结果      │
└────────┬────────┘  └────────┬────────┘  └────────┬────────┘
         │                    │                    │
         └────────────────────┼────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 4. Master Agent 整合                                                        │
│    • 收集所有 Specialist 结果                                               │
│    • 构建整合提示词                                                          │
│    • 调用 LLM 生成最终答案                                                   │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 5. 返回响应                                                                  │
│    • 最终答案                                                                │
│    • 执行步骤记录                                                            │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. 知识库数据流

### 4.1 文件上传流程

```
用户上传文件 (PDF/Word/MD/TXT/JSON/CSV)
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 1. Gateway                                                                   │
│    • 检查文件类型                                                            │
│    • 检查文件大小                                                             │
│    • 提取 tenant_id                                                          │
│    • 转发到 Knowledge Service                                                │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 2. Knowledge Service - 创建文档                                             │
│    • 生成 document_id                                                        │
│    • 保存文件到临时目录                                                       │
│    • 创建文档记录 (SQLite)                                                   │
│    • 状态: processing                                                       │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 3. 解析文件                                                                  │
│    ┌─────────────────────────────────────────────────────────────────────┐  │
│    │ PDF:  pdf_parser.go  → pdftotext / pdfplumber                       │  │
│    │ Word: docx_parser.go → docx extraction                              │  │
│    │ MD:   md_parser.go   → 直接读取                                      │  │
│    │ TXT:  txt_parser.go  → 直接读取                                      │  │
│    │ JSON: json_parser.go → 解析 + 提取 text 字段                         │  │
│    │ CSV:  csv_parser.go  → 逐行读取 + 指定列                             │  │
│    └─────────────────────────────────────────────────────────────────────┘  │
│    • 提取纯文本内容                                                          │
│    • 提取元数据 (标题、作者、创建时间等)                                       │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 4. 智能分块                                                                  │
│    ┌─────────────────────────────────────────────────────────────────────┐  │
│    │ 根据策略选择分块器:                                                   │  │
│    │                                                                       │  │
│    │ Token 分块:                                                           │  │
│    │   • 估算 token 数 (中文 1:1, 英文 4:1)                                │  │
│    │   • 找边界 (段落 > 句子 > 标点)                                       │  │
│    │   • chunk_size: 512, overlap: 50                                     │  │
│    │                                                                       │  │
│    │ Semantic 分块:                                                        │  │
│    │   • 分句                                                              │  │
│    │   • 批量 Embedding                                                    │  │
│    │   • 计算相邻句子相似度                                                 │  │
│    │   • 相似度 < 0.45 = 话题边界                                          │  │
│    │                                                                       │  │
│    │ Markdown 分块:                                                        │  │
│    │   • 按 # 标题分割                                                     │  │
│    │   • 保持层级结构                                                      │  │
│    └─────────────────────────────────────────────────────────────────────┘  │
│    • 输出: []Chunk{ID, Content, Index}                                      │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 5. 批量向量化                                                                │
│    • 分批调用 Embedding API (每批 20 个)                                     │
│    • 记录 Token 消耗                                                        │
│    • 重试失败的批次                                                         │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ├──────────────────────────────────────────────────┐
    │                                                  │
    ▼                                                  ▼
┌─────────────────────────┐                ┌─────────────────────────┐
│ 6a. 存入 Qdrant         │                │ 6b. 存入 MongoDB        │
│                         │                │                         │
│ • collection: chunks    │                │ • collection: chunks    │
│ • id: chunk_id          │                │ • doc_id: document_id   │
│ • vector: []float64     │                │ • content: string       │
│ • payload:              │                │ • chunk_index: int      │
│   - content             │                │ • metadata: {}          │
│   - document_id         │                │                         │
│   - chunk_index         │                │ (用于 BM25 检索)        │
└─────────────────────────┘                └─────────────────────────┘
    │                                                  │
    └──────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 7. 更新文档状态                                                              │
│    • 状态: ready                                                            │
│    • chunk_count: N                                                         │
│    • 保存元数据 (SQLite)                                                     │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 8. 返回响应                                                                  │
│    • document_id                                                            │
│    • chunk_count                                                            │
│    • chunk_ids                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

```

    │                                                     │
    └─────────────────────┬───────────────────────────────┘
                          │
                          ▼

│                                                                             │

│ BM25结果: [B:0.92, A:0.88, D:0.82, E:0.78]                                  │
│                                                                             │

│   A: 1/(60+1) + 1/(60+2) = 0.0164 + 0.0161 = 0.0325                       │

│   E: 1/(60+4) = 0.0156                                                    │
│                                                                             │
│ 排序后: [A:0.0325, B:0.0325, D:0.0315, C:0.0159, E:0.0156]                 │

    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 2. 后处理                                                                    │
│    • 过滤低分结果 (score < threshold)                                        │
│    • 填充完整内容 (从 MongoDB)                                               │
│    • 添加元数据                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 3. 返回响应                                                                  │
│    results: [                                                               │
│      {chunk_id, document_id, content, score, metadata},                    │
│      ...                                                                   │
│    ]                                                                       │
```

## 6. Harness 治理数据流

### 6.1 Gate 检查流程

```
请求进入 Harness Service
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ Gate 1: 输入护栏 (guardrail.go)                                              │
│                                                                             │
│ 检查项:                                                                      │
│   • Prompt Injection 检测 (关键词 + 正则)                                    │
│   • 敏感信息检测 (API Key、密码、私钥)                                       │
│   • 自定义规则检测                                                          │
│                                                                             │
│ 输入: content (用户消息)                                                     │
```

### 7.2 数据一致性策略

| 场景 | 策略 |
|------|------|
| 文档上传 | 最终一致性：先写入 Qdrant，再写入 MongoDB，最后更新 SQLite |
| 检索 | 读优先：优先从 Qdrant 读取，失败则回退到 MongoDB |
| 会话 | 强一致性：SQLite WAL 模式保证 |
| 记忆 | 最终一致性：先写入 Qdrant，再写入 SQLite |
